Generate and return the Empirical Cumulative Distribution Function (ECDF) of the input univariate sample.  




\doxysubsection{Detailed Description}
Generate and return the Empirical Cumulative Distribution Function (ECDF) of the input univariate sample. 


\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ in}}  & {\em Sample} & \+: The contiguous array of shape {\ttfamily (np)} (rank {\ttfamily 1}) of type {\ttfamily real} of kind \mbox{\hyperlink{namespaceConstants__mod_aaf5c6c840ed0befe5a37da626c35716f}{RK32}}, \mbox{\hyperlink{namespaceConstants__mod_a855643f887b4c787e6542e9e7849680f}{RK64}}, or \mbox{\hyperlink{namespaceConstants__mod_aeef7c44b90b75aeeebdef1da504886cb}{RK128}} containing the univariate sample, where {\ttfamily np} is the number of data points in the sample. \\
\hline
\mbox{\texttt{ in}}  & {\em Weight} & \+: The contiguous vector of length {\ttfamily np}, of either type {\ttfamily real} of the same kind as the input {\ttfamily Sample} or type {\ttfamily integer} of kind \mbox{\hyperlink{namespaceConstants__mod_ae3573d46486b5f1a0a8d5033a9d12879}{IK8}}, \mbox{\hyperlink{namespaceConstants__mod_acd3c33e94d12f6e7d7c0ee79b2d6280b}{IK16}}, \mbox{\hyperlink{namespaceConstants__mod_ab12a86d74008f0d7f287c3564c3defd9}{IK32}}, or \mbox{\hyperlink{namespaceConstants__mod_ad6aaf2f477491dc84e7ebbab3b64694e}{IK64}}, containing the corresponding weights of the data points in the input {\ttfamily Sample}. While the elements of {\ttfamily Weight} can be negative or zero, note that {\ttfamily Sample} elements with non-\/positive weights will be ignored ({\bfseries{optional}}, default = a vector of ones).\\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
{\ttfamily ECDF} \+: The output array of of the same shape, length, type, and kind as the input {\ttfamily Sample}, containing the Empirical Cumulative Distribution Function (ECDF) of the input univariate {\ttfamily Sample}.
\end{DoxyReturn}
\begin{DoxyRemark}{Remarks}
This procedure is {\ttfamily pure}.
\end{DoxyRemark}
\begin{DoxyNote}{Note}
If the input sample is to be an array of type {\ttfamily integer}, simply convert the sample to an array of type {\ttfamily real} of the desired kind for the output {\ttfamily real} mean of the sample. There is no point in accepting an input sample of type {\ttfamily integer} since it will be inevitably converted to an array of type {\ttfamily real} within the procedure to avoid potential integer overflow. Furthermore, an {\ttfamily Sample} of type {\ttfamily integer} creates ambiguity about the {\ttfamily kind} of the {\ttfamily real}-\/valued returned mean by the procedure.
\end{DoxyNote}
