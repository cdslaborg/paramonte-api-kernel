\mbox{\texttt{ in}}  & {\em Index} & \+: The input contiguous array of shape {\ttfamily (\+:)} of type {\ttfamily integer} of default kind \mbox{\hyperlink{namespaceConstants__mod_a2bf4d6767c9954d3a495d10ab6b538d6}{IK}}, containing the indices of the input {\ttfamily Array} where {\ttfamily Selection} must be selected to construct {\ttfamily Array\+New}.~\newline
 ~ All elements of {\ttfamily Index} must have values between {\ttfamily 1} and the length of {\ttfamily Array} plus {\ttfamily 1}.~\newline

\mbox{\texttt{ in}}  & {\em positive} & \+: The input {\ttfamily logical} of default kind \mbox{\hyperlink{namespaceConstants__mod_a68ac316f10b57be996670bc7c0abed9d}{LK}} indicating whether the elements of {\ttfamily Index} are all positive (indicating counts from the beginning of {\ttfamily Array}). Setting {\ttfamily positive = .true.} will lead to a slightly better runtime performance of the algorithm since a conversion of potentially-\/negative index values to the corresponding positive values (counting from the beginning of {\ttfamily Array} will be avoided ({\bfseries{optional}}, default = {\ttfamily .false.}). \\
\hline
\mbox{\texttt{ in}}  & {\em sorted} & \+: The input {\ttfamily logical} of default kind \mbox{\hyperlink{namespaceConstants__mod_a68ac316f10b57be996670bc7c0abed9d}{LK}} indicating whether the selections of the specified input {\ttfamily Index} are all ascending-\/ordered. This includes the negative values of {\ttfamily Index} {\bfseries{after}} they are converted to the corresponding {\bfseries{positive}} indices from the beginning of the input {\ttfamily Array}. Setting {\ttfamily sorted = .true.} will lead to a better runtime performance of the algorithm since a call to the \mbox{\hyperlink{interfaceArraySort__mod_1_1sort}{sort}} to sort the index values in ascending order will be avoided. However, the onus will be on the user to guarantee the ascending order of the elements of the input Index ({\bfseries{optional}}, default = {\ttfamily .false.}).\\
