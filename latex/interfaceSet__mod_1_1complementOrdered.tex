\doxysubsection{Detailed Description}
Generate and return an array of elements of {\ttfamily Ordered\+Super\+Set} that are not in {\ttfamily Ordered\+Set}. This is the same as the ordinary {\ttfamily complement()} algorithm, however, the orderliness of the input sets leads to faster runtime. 


\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ in}}  & {\em Ordered\+Set} & \+: The input set whose elements will be subtracted from the {\ttfamily Ordered\+Super\+Set}. \\
\hline
\mbox{\texttt{ in}}  & {\em Ordered\+Super\+Set} & \+: The input {\ttfamily Ordered\+Super\+Set} whose missing elements in {\ttfamily Ordered\+Set} will be returned.\\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
{\ttfamily Complement} \+: The array of elements of {\ttfamily Ordered\+Set} that are not in {\ttfamily Ordered\+Super\+Set}. By definition, {\ttfamily size(\+Complement) = size(\+Ordered\+Super\+Set) -\/ size(\+Ordered\+Set)}.
\end{DoxyReturn}
\begin{DoxyWarning}{Warning}
All elements of {\ttfamily Ordered\+Set} and {\ttfamily Ordered\+Super\+Set} must be {\bfseries{unique and ordered ascending}}. The unique elements of any arbitrary vector can be obtained by passing the array to \mbox{\hyperlink{interfaceUnique__mod_1_1genUnique}{Unique\+\_\+mod\+::gen\+Unique()}}.

It is assumed that {\ttfamily Ordered\+Set} is subset of {\ttfamily Ordered\+Super\+Set}. This condition will NOT be checked in this procedure. If this condition does not hold, the results will not be accurate.
\end{DoxyWarning}
