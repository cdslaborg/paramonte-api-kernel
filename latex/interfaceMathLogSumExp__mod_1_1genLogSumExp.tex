\doxysubsection{Detailed Description}
Generate and return the natural logarithm of the sum of the exponential of the input {\ttfamily Array} robustly (without numerical overflow). 


\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ in}}  & {\em Array} & \+: The input contiguous array of shape {\ttfamily (\+:)} of either ~\newline
 \&bsp; type {\ttfamily real} of kind \mbox{\hyperlink{namespaceConstants__mod_aaf5c6c840ed0befe5a37da626c35716f}{RK32}}, \mbox{\hyperlink{namespaceConstants__mod_a855643f887b4c787e6542e9e7849680f}{RK64}}, or \mbox{\hyperlink{namespaceConstants__mod_aeef7c44b90b75aeeebdef1da504886cb}{RK128}} or, ~\newline

 whose natural logarithm of sum of the exponential of its elements will be generated and returned. \\
\hline
\mbox{\texttt{ in}}  & {\em max\+Array} & \+: The input scalar of the same type and kind as the input {\ttfamily Array} representing the maximum value in {\ttfamily Array} (i.\+e., {\ttfamily max\+Array = maxval(\+Array)}). If the input {\ttfamily Array} is of {\ttfamily complex} type. \\
\hline
\mbox{\texttt{ in}}  & {\em cenabled} & \+: The input scalar of type {\ttfamily logical} of default kind. If {\ttfamily .true.}, then runtime checks for underflow occurrence will be enabled (hence the name {\ttfamily cenabled} standing for {\itshape Checks ENABLED}). Enabling this option can aid runtime efficiency when the division of a significant number of elements of {\ttfamily Array} (for example, half or more) by {\ttfamily max\+Array} causes underflow. In such cases, the exponentiation is avoided if {\ttfamily cenabled = .true.} leading to faster runtime by avoiding exponentiation since it is highly expensive (on the order of $\sim$200 CPU cycles). See \href{\#benchmark-genLogSumExp}{\texttt{ the relevant benchmark here}}. ({\bfseries{optional}}, default = {\ttfamily .false.}).\\
\hline

\begin{DoxyWarning}{Warning}
The input value of {\ttfamily max\+Array} must represent the maximum value among the elements of the input {\ttfamily Array}, that is, {\ttfamily real(max\+Array) == maxval(real(\+Array))} must hold. The validity of this is only checked if the preprocessor macro {\ttfamily CHECK\+\_\+\+ENABLED=1} is set at build time.

This procedure is {\ttfamily impure} when the library is compiled with macro {\ttfamily CHECK\+\_\+\+ENABLED=1}. By default, it is {\ttfamily pure} in {\ttfamily release} build and {\ttfamily impure} in {\ttfamily debug} and {\ttfamily testing} builds.

\begin{DoxyEnumerate}
\item If the input array has many (half the size of array or more) elements whose division by the {\ttfamily maxval(\+Array)} causes underflow, then setting {\ttfamily cenabled = .true.} when calling \mbox{\hyperlink{interfaceMathLogSumExp__mod_1_1genLogSumExp}{gen\+Log\+Sum\+Exp}} will likely result in a faster runtime. Conversely, if the divisions are not expected to cause any or too many underflows, then set {\ttfamily cenabled = .true.} to improve cache coherence and runtime performance (at the expense of occasional expensive but redundant exponentiations).
\item Note the performance of manual computation against \mbox{\hyperlink{interfaceMathLogSumExp__mod_1_1genLogSumExp}{gen\+Log\+Sum\+Exp}} with or without setting the optional argument {\ttfamily cenabled}.
