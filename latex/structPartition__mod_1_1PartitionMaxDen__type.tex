\doxysubsection{Detailed Description}
The {\ttfamily \mbox{\hyperlink{structPartition__mod_1_1PartitionMaxDen__type}{Partition\+Max\+Den\+\_\+type}}} class. 

This function can serve as the constructor for {\ttfamily \mbox{\hyperlink{structPartition__mod_1_1PartitionMaxDen__type}{Partition\+Max\+Den\+\_\+type}}} while also performing the partitioning for the first time. Construct a hierarchical partitioning of the input {\ttfamily Point} using a combination of Kmeans, Kvolume, and density maximization algorithms.


\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ in}}  & {\em Point} & \+: An array of size {\ttfamily (nd = \#dimensions, np = \#observations)} containing the input set of points to partition. \\
\hline
\mbox{\texttt{ in}}  & {\em parent\+Log\+Vol\+Normed} & \+: The logarithm of the volume of the input set of points normalized to the volume of the unit {\ttfamily nd}-\/ball. \\
\hline
\mbox{\texttt{ in}}  & {\em nc} & \+: The number of clusters at each level of partitioning ({\bfseries{optional}}, default = {\ttfamily 2}). \\

\hline

\hline
\mbox{\texttt{ in}}  & {\em min\+Size} & \+: The minimum allowed size of a partition, that is, the minimum allowed number of points in a partition ({\bfseries{optional}}, default = {\ttfamily 0}). \\

\mbox{\texttt{ in}}  & {\em stan\+Enabled} & \+: A logical value, indicating weather {\ttfamily Point} should be standardized prior to each Kmeans clustering ({\bfseries{optional}}, default = {\ttfamily .true.}). \\

\mbox{\texttt{ in}}  & {\em mahal\+Sq\+Weight\+Exponent} & \+: The exponent with which the density of the partitions are exponentiated to be used as the Mahalanobis distance weights ({\bfseries{optional}}, default = {\ttfamily 0.}). \\

\begin{DoxyWarning}{Warning}
On output, {\ttfamily Point} will be reordered.

When {\ttfamily trim\+Enabled = .false.}, only the first {\ttfamily 1\+:ne} elements of all allocatable arrays contain meaningful information. As such, the rest of the trailing elements must be properly ignored when using the allocatable Partition components. For example, upon exit, only {\ttfamily PartitionLog\+Vol\+Normed(1\+:Partitionne)} contain meaningful information.

When {\ttfamily trim\+Enabled = .true.}, all allocatable arrays will be trimmed from size {\ttfamily nemax} to the size {\ttfamily ne}. In such a case, calling the method {\ttfamily Partitiontun()} after constructing the {\ttfamily Partition} object for the first time will likely lead to segmentation fault error.
\end{DoxyWarning}
\begin{DoxyRemark}{Remarks}
If no input value if provided for {\ttfamily parent\+Log\+Vol\+Normed}, both {\ttfamily parent\+Log\+Vol\+Normed} and {\ttfamily point\+Log\+Vol\+Normed} will (must) be set to {\ttfamily NEGINF\+\_\+\+RK}.
\end{DoxyRemark}
