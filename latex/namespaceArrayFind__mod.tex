{\bfseries{Benchmark moral}} \href{\#moralb-findLoc-sortedPositive_vs_defaultOptions}{\texttt{ ⛓}} \DoxyHorRuler{0}

\begin{DoxyEnumerate}
\item The procedures under the generic interface \mbox{\hyperlink{interfaceArrayFind__mod_1_1findLoc}{find\+Loc}} take two optional {\ttfamily sorted} and {\ttfamily positive} input arguments when the input array of instances {\ttfamily Instance} is also specified. Setting {\ttfamily sorted = .true., positive = .true.} guarantees to the procedures that all elements of {\ttfamily Instance} are sorted in {\bfseries{ascending-\/order}} and that no negative or zero value is present in {\ttfamily Instance}. Hence, the maximum value in {\ttfamily Instance} corresponds to {\ttfamily Instance(size(\+Instance))}, saving the procedures the time to find the maximum value of the input {\ttfamily Instance}.
\item As evidenced by the above benchmark, when {\ttfamily sorted = .true., positive = .true.} can be guaranteed by the user, the runtime performance of the procedures could possibly increase {\bfseries{2-\/3 times}} compared to when the default values have to be assumed, corresponding to the worst case scenario ({\ttfamily sorted = .false., positive = .false.}).
\item The results of this benchmark equally hold for the functions under the generic interface \mbox{\hyperlink{interfaceArrayFind__mod_1_1genLoc}{gen\+Loc}}.
\end{DoxyEnumerate}
