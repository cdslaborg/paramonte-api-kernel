\doxysubsection{Detailed Description}
Copy the lower-\/triangle (excluding diagonals) of the input {\ttfamily from} square matrix to the lower-\/triangle of the input matrix {\ttfamily to}. 

This subroutine interface is a shortcut to avoid loops for copying the lower triangle of a matrix to another existing one. While writing the loop is simple, since it occurs frequently, these subroutines provide a cleaner solution.


\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ in}}  & {\em from} & \+: The contiguous square matrix whose lower-\/triangle will have to be copied into the destination matrix. All elements of {\ttfamily from} shall be all reals of either \mbox{\hyperlink{namespaceConstants__mod_aaf5c6c840ed0befe5a37da626c35716f}{RK32}}, \mbox{\hyperlink{namespaceConstants__mod_a855643f887b4c787e6542e9e7849680f}{RK64}}, \mbox{\hyperlink{namespaceConstants__mod_aeef7c44b90b75aeeebdef1da504886cb}{RK128}} kinds. \\
\hline
\mbox{\texttt{ in,out}}  & {\em to} & \+: The destination matrix of the same type and kind as {\ttfamily from} to which the lower-\/triangle of {\ttfamily from} will be copied.\\
\hline
\end{DoxyParams}
\begin{DoxyWarning}{Warning}
The input matrices {\ttfamily from} and {\ttfamily to} must be square. This condition is checked only when the preprocessor macro {\ttfamily CHECK\+\_\+\+ENABLED} evaluates to true.

This procedure is {\ttfamily impure} when the library is compiled with macro {\ttfamily CHECK\+\_\+\+ENABLED=1}. By default, it is {\ttfamily pure} in {\ttfamily release} build and {\ttfamily impure} in {\ttfamily debug} and {\ttfamily testing} builds.
\end{DoxyWarning}
