\doxysubsection{Detailed Description}
The {\ttfamily \mbox{\hyperlink{structPartition__mod_1_1PartitionMinVol__type}{Partition\+Min\+Vol\+\_\+type}}} class. 

This function can serve as the constructor for {\ttfamily \mbox{\hyperlink{structPartition__mod_1_1PartitionMinVol__type}{Partition\+Min\+Vol\+\_\+type}}} while also performing the partitioning for the first time. Construct a hierarchical partitioning of the input {\ttfamily Point} using a combination of Kmeans, Kvolume, and density maximization algorithms.

\begin{DoxyWarning}{Warning}
On output, {\ttfamily Point} will be reordered.

When {\ttfamily trim\+Enabled = .false.}, only the first {\ttfamily 1\+:ne} elements of all allocatable arrays contain meaningful information. As such, the rest of the trailing elements must be properly ignored when using the allocatable Partition components. For example, upon exit, only {\ttfamily PartitionLog\+Vol\+Normed(1\+:Partitionne)} contain meaningful information.

When {\ttfamily trim\+Enabled = .true.}, all allocatable arrays will be trimmed from size {\ttfamily nemax} to the size {\ttfamily ne}. In such a case, calling the method {\ttfamily Partitiontun()} after constructing the {\ttfamily Partition} object for the first time will likely lead to segmentation fault error.
\end{DoxyWarning}
\begin{DoxyRemark}{Remarks}
If no input value if provided for {\ttfamily parent\+Log\+Vol\+Normed}, both {\ttfamily parent\+Log\+Vol\+Normed} and {\ttfamily point\+Log\+Vol\+Normed} will (must) be set to {\ttfamily NEGINF\+\_\+\+RK}.
\end{DoxyRemark}
