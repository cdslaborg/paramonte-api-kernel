\begin{DoxyWarning}{Warning}
If the length of the input {\ttfamily Array} is longer than the largest number that can be represented with an {\ttfamily integer} of kind \mbox{\hyperlink{namespaceConstants__mod_ab12a86d74008f0d7f287c3564c3defd9}{IK32}}, then, the length of array must be provided as the input {\ttfamily integer} argument {\ttfamily len\+Array} of kind \mbox{\hyperlink{namespaceConstants__mod_ad6aaf2f477491dc84e7ebbab3b64694e}{IK64}}.

The procedures under this generic interface are all {\ttfamily pure} except when the input argument {\ttfamily \mbox{\hyperlink{interfaceArraySort__mod_1_1isSorted}{is\+Sorted}}} is present.
\end{DoxyWarning}
\begin{DoxyNote}{Note}
The procedures under this generic interface sort the input array by a mixture of Quicksort and selection sorting methods. When the size the array to be sorted reaches {\ttfamily 30} or less, the algorithm switches from Quicksort to selection sorting.

In almost all scenarios, this algorithm must be the default choice for any sorting.

This algorithm is typically of order $ N \log_2 N$, and the worst-\/case order of $ N^2$. The worst case performance occurs for completely sorted input arrays.

To sort the input array in descending order, simply call the output array elements from the end to the beginning or, rewrite the array with {\ttfamily Array = Array( ubound(Array, dim = 1) \+: lbound(Array, dim = 1) \+: -\/1 )}.

A rooter array can be sorted along with a leader array with the help of \mbox{\hyperlink{interfaceArraySort__mod_1_1sortIndex}{sort\+Index}}.
\end{DoxyNote}
