\begin{DoxyWarning}{Warning}
If the length of the input {\ttfamily Array} is longer than the largest number that can be represented with an {\ttfamily integer} of kind \mbox{\hyperlink{namespaceConstants__mod_ab12a86d74008f0d7f287c3564c3defd9}{IK32}}, then, the length of array must be provided as the input {\ttfamily integer} argument {\ttfamily len\+Array} of kind \mbox{\hyperlink{namespaceConstants__mod_ad6aaf2f477491dc84e7ebbab3b64694e}{IK64}}.

This procedure is {\ttfamily impure} when the library is compiled with macro {\ttfamily CHECK\+\_\+\+ENABLED=1}. By default, it is {\ttfamily pure} in {\ttfamily release} build and {\ttfamily impure} in {\ttfamily debug} and {\ttfamily testing} builds.
\end{DoxyWarning}
\begin{DoxyNote}{Note}
The procedures under this generic interface sort the input array by a mixture of Quicksort and straight insertion sorting methods. When the size the array to be sorted reaches {\ttfamily 30} or less, the algorithm switches from Quicksort to straight insertion sorting.

In almost all scenarios, this algorithm must be the default choice for any array index sorting.

This algorithm is typically of order $ N \log_2 N$, and the worst-\/case order of $ N^2$. The worst case performance occurs for completely sorted input arrays.

To sort the input array in descending order, simply call the output {\ttfamily Index} elements from the end to the beginning or, rewrite the array with {\ttfamily Array = Array( Index( ubound(Array, dim = 1) \+: lbound(Array, dim = 1) \+: -\/1 ) )}.

A rooter array can be sorted along with a leader array with the help of \mbox{\hyperlink{interfaceSort__mod_1_1sortIndex}{sort\+Index}}.
\end{DoxyNote}
