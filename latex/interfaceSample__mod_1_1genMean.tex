Generate and return the (weighted) mean of the input {\ttfamily Sample}, optionally weighted by the input {\ttfamily Weight}.  




\doxysubsection{Detailed Description}
Generate and return the (weighted) mean of the input {\ttfamily Sample}, optionally weighted by the input {\ttfamily Weight}. 


\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ in}}  & {\em Sample} & \+: The contiguous array of {\ttfamily real} type of kind \mbox{\hyperlink{namespaceConstants__mod_aaf5c6c840ed0befe5a37da626c35716f}{RK32}}, \mbox{\hyperlink{namespaceConstants__mod_a855643f887b4c787e6542e9e7849680f}{RK64}}, or \mbox{\hyperlink{namespaceConstants__mod_aeef7c44b90b75aeeebdef1da504886cb}{RK128}} of shape {\ttfamily (np)}, {\ttfamily (nd,np)} or {\ttfamily (np,nd)} containing the sample, where {\ttfamily nd} is the number of attributes of data in the sample and {\ttfamily np} is the number of data points in the sample. \\
\hline
\mbox{\texttt{ in}}  & {\em Weight} & \+: The contiguous vector of length {\ttfamily np}, of either type {\ttfamily real} of the same kind as the input {\ttfamily Sample} or type {\ttfamily integer} of kind \mbox{\hyperlink{namespaceConstants__mod_ae3573d46486b5f1a0a8d5033a9d12879}{IK8}}, \mbox{\hyperlink{namespaceConstants__mod_acd3c33e94d12f6e7d7c0ee79b2d6280b}{IK16}}, \mbox{\hyperlink{namespaceConstants__mod_ab12a86d74008f0d7f287c3564c3defd9}{IK32}}, or \mbox{\hyperlink{namespaceConstants__mod_ad6aaf2f477491dc84e7ebbab3b64694e}{IK64}}, containing the corresponding weights of the data points in the input {\ttfamily Sample} ({\bfseries{optional}}, default = a vector of ones). \\
\hline
\mbox{\texttt{ in}}  & {\em dim} & \+: An {\ttfamily integer} of default kind \mbox{\hyperlink{namespaceConstants__mod_a2bf4d6767c9954d3a495d10ab6b538d6}{IK}} representing the dimension (1 or 2) along which data is stored in {\ttfamily Sample}. For example, {\ttfamily dim = 1} means that {\ttfamily Sample} has the shape {\ttfamily (np,nd)}, whereas {\ttfamily dim = 2} implies a {\ttfamily (nd,np)} shape. This input argument is available only when the input {\ttfamily Sample} has more than one dimension.\\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
{\ttfamily Mean} \+: The output mean of the same type and kind as the input {\ttfamily Sample}.~\newline
 When the input {\ttfamily Sample} has the shape {\ttfamily (np)}, the output mean is a scalar.~\newline
 When the input {\ttfamily Sample} has the shape {\ttfamily (np)}, the output mean is a vector of length {\ttfamily nd}.
\end{DoxyReturn}
\begin{DoxyWarning}{Warning}
Note the default assumed shape and size of the input argument {\ttfamily Sample} when it is a matrix of shape {\ttfamily (\+:,\+:)}.

This procedure is {\ttfamily impure} when the library is compiled with macro {\ttfamily CHECK\+\_\+\+ENABLED=1}. By default, it is {\ttfamily pure} in {\ttfamily release} build and {\ttfamily impure} in {\ttfamily debug} and {\ttfamily testing} builds.
\end{DoxyWarning}
\begin{DoxyNote}{Note}
If the input sample is to be an array of type {\ttfamily integer}, simply convert the sample to an array of type {\ttfamily real} of the desired kind for the output {\ttfamily real} mean of the sample. There is no point in accepting an input sample of type {\ttfamily integer} since it will be inevitably converted to an array of type {\ttfamily real} within the procedure to avoid potential integer overflow. Furthermore, an {\ttfamily Sample} of type {\ttfamily integer} creates ambiguity about the {\ttfamily kind} of the {\ttfamily real}-\/valued returned mean by the procedure.

Note that the mean of any one or two-\/dimensional sample can be simply computed via the Fortran intrinsic routine {\ttfamily \mbox{\hyperlink{namespaceMisc__mod_a715e59341730f07df7823e76c513d837}{sum()}}}\+:

\mbox{\texttt{ in}}  & {\em Sample} & \+: The contiguous array of {\ttfamily real} type of kind \mbox{\hyperlink{namespaceConstants__mod_aaf5c6c840ed0befe5a37da626c35716f}{RK32}}, \mbox{\hyperlink{namespaceConstants__mod_a855643f887b4c787e6542e9e7849680f}{RK64}}, or \mbox{\hyperlink{namespaceConstants__mod_aeef7c44b90b75aeeebdef1da504886cb}{RK128}} of shape {\ttfamily (np)}, {\ttfamily (nd,np)} or {\ttfamily (np,nd)} containing the sample, where {\ttfamily nd} is the number of attributes of data in the sample and {\ttfamily np} is the number of data points in the sample. \\
\hline
\mbox{\texttt{ in}}  & {\em Weight} & \+: The contiguous vector of length {\ttfamily np}, of either type {\ttfamily real} of the same kind as the input {\ttfamily Sample} or type {\ttfamily integer} of kind \mbox{\hyperlink{namespaceConstants__mod_ae3573d46486b5f1a0a8d5033a9d12879}{IK8}}, \mbox{\hyperlink{namespaceConstants__mod_acd3c33e94d12f6e7d7c0ee79b2d6280b}{IK16}}, \mbox{\hyperlink{namespaceConstants__mod_ab12a86d74008f0d7f287c3564c3defd9}{IK32}}, or \mbox{\hyperlink{namespaceConstants__mod_ad6aaf2f477491dc84e7ebbab3b64694e}{IK64}}, containing the corresponding weights of the data points in the input {\ttfamily Sample} ({\bfseries{optional}}, default = a vector of ones). \\
\hline
\mbox{\texttt{ in}}  & {\em dim} & \+: An {\ttfamily integer} of default kind \mbox{\hyperlink{namespaceConstants__mod_a2bf4d6767c9954d3a495d10ab6b538d6}{IK}} representing the dimension (1 or 2) along which data is stored in {\ttfamily Sample}. For example, {\ttfamily dim = 1} means that {\ttfamily Sample} has the shape {\ttfamily (np,nd)}, whereas {\ttfamily dim = 2} implies a {\ttfamily (nd,np)} shape. This input argument is available only when the input {\ttfamily Sample} has more than one dimension.\\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
{\ttfamily Mean} \+: The output mean of the same type and kind as the input {\ttfamily Sample}.~\newline
 When the input {\ttfamily Sample} has the shape {\ttfamily (np)}, the output mean is a scalar.~\newline
 When the input {\ttfamily Sample} has the shape {\ttfamily (np)}, the output mean is a vector of length {\ttfamily nd}.
\end{DoxyReturn}
\begin{DoxyWarning}{Warning}
Note the default assumed shape and size of the input argument {\ttfamily Sample} when it is a matrix of shape {\ttfamily (\+:,\+:)}.

This procedure is {\ttfamily impure} when the library is compiled with macro {\ttfamily CHECK\+\_\+\+ENABLED=1}. By default, it is {\ttfamily pure} in {\ttfamily release} build and {\ttfamily impure} in {\ttfamily debug} and {\ttfamily testing} builds.
\end{DoxyWarning}
\begin{DoxyNote}{Note}
If the input sample is to be an array of type {\ttfamily integer}, simply convert the sample to an array of type {\ttfamily real} of the desired kind for the output {\ttfamily real} mean of the sample. There is no point in accepting an input sample of type {\ttfamily integer} since it will be inevitably converted to an array of type {\ttfamily real} within the procedure to avoid potential integer overflow. Furthermore, an {\ttfamily Sample} of type {\ttfamily integer} creates ambiguity about the {\ttfamily kind} of the {\ttfamily real}-\/valued returned mean by the procedure.

Note that the mean of any one or two-\/dimensional sample can be simply computed via the Fortran intrinsic routine {\ttfamily \mbox{\hyperlink{namespaceMisc__mod_a715e59341730f07df7823e76c513d837}{sum()}}}\+:
