\doxysubsection{Detailed Description}
Return a sample of shape {\ttfamily (np)}, or {\ttfamily (nd,np)} or {\ttfamily (np,nd)} that is shifted by the specified input {\ttfamily Amount} along the (optionally) specified axis {\ttfamily dim}. Here, {\ttfamily nd} stands for the number of dimensions (data attributes) of the input {\ttfamily Sample} and {\ttfamily np} represents the number of data points in the {\ttfamily Sample}. If the input {\ttfamily Amount} is the negative of the mean of the sample, then the returned sample will have a mean of zero. 


\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ in,out}}  & {\em Sample} & \+: The contiguous array of type {\ttfamily real} of kind \mbox{\hyperlink{namespaceConstants__mod_aaf5c6c840ed0befe5a37da626c35716f}{RK32}}, \mbox{\hyperlink{namespaceConstants__mod_a855643f887b4c787e6542e9e7849680f}{RK64}}, or \mbox{\hyperlink{namespaceConstants__mod_aeef7c44b90b75aeeebdef1da504886cb}{RK128}} of shape {\ttfamily (np)}, {\ttfamily (nd,np)}, or {\ttfamily (np,nd)} containing the sample. On output, it is completely overwritten with the shifted sample. \\
\hline
\mbox{\texttt{ in}}  & {\em Amount} & \+: The input scalar or the contiguous vector of the same type and kind as {\ttfamily Sample} representing the amount by which the input sample must be shifted.~\newline
 If the input {\ttfamily Sample} is a 1D array, then {\ttfamily Amount} must be a scalar.~\newline
 If the input {\ttfamily Sample} is a 2D array, then {\ttfamily Amount} must be a vector of size {\ttfamily nd}.~\newline
 If the sample is to be shifted toward the origin to have a mean of zero, then the input {\ttfamily Amount} corresponds to the negative of the current mean of the sample that is returned by procedures collectively represented by the generic interface \mbox{\hyperlink{interfaceSampleMean__mod_1_1genMean}{gen\+Mean}}.~\newline
 Note that the size of the input {\ttfamily Amount} must be consistent with the size of the input {\ttfamily Sample}.~\newline
 If the input argument {\ttfamily dim = 2} (default) then, {\ttfamily size(\+Amount) == size(Sample,dim=2)} must hold.~\newline
 If the input argument {\ttfamily dim = 1} (default) then, {\ttfamily size(\+Amount) == size(Sample,dim=1)} must hold.~\newline
 \\
\hline
\mbox{\texttt{ in}}  & {\em dim} & \+: An integer of default kind \mbox{\hyperlink{namespaceConstants__mod_a2bf4d6767c9954d3a495d10ab6b538d6}{IK}} indicating which dimension of the input {\ttfamily Sample} iterates over the individual data points.~\newline
 If {\ttfamily dim = 1} or {\ttfamily dim /= 2}, the input {\ttfamily Sample} is assumed to have the shape {\ttfamily (np,nd)}.~\newline
 If {\ttfamily dim = 2}, the input {\ttfamily Sample} is assumed to have the shape {\ttfamily (nd,np)}.~\newline
 ({\bfseries{optional}}, default = {\ttfamily 2}. {\bfseries{This input argument is available only if the input {\ttfamily Sample} is a 2D array}}.)\\

\begin{DoxyRemark}{Remarks}
The procedures represented by this generic interface have the same functionality as \mbox{\hyperlink{interfaceSampleShift__mod_1_1genShifted}{gen\+Shifted}}. However, unlike \mbox{\hyperlink{interfaceSampleShift__mod_1_1genShifted}{gen\+Shifted}} which are functions and return the shifted data as a new allocatable object, \mbox{\hyperlink{interfaceSampleShift__mod_1_1getShifted}{get\+Shifted}} subroutines shift the input data {\bfseries{in place}}, making it potentially more efficient and less memory-\/consuming than \mbox{\hyperlink{interfaceSampleShift__mod_1_1genShifted}{gen\+Shifted}} functions.
\end{DoxyRemark}
