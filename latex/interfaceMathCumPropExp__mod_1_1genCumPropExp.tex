\doxysubsection{Detailed Description}
Generate and return the cumulative sum of the proportions of the exponential of the input array, optionally in the backward direction and, optionally reverse the output cumulative sum upon return. The returned array is normalized such that all of its elements fall in the range $[0,1]$. All operations are performed while avoiding arithmetic overflow. 


\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ in}}  & {\em Array} & \+: The input contiguous array of shape {\ttfamily (\+:)} of type {\ttfamily real} of kind \mbox{\hyperlink{namespaceConstants__mod_aaf5c6c840ed0befe5a37da626c35716f}{RK32}}, \mbox{\hyperlink{namespaceConstants__mod_a855643f887b4c787e6542e9e7849680f}{RK64}}, or \mbox{\hyperlink{namespaceConstants__mod_aeef7c44b90b75aeeebdef1da504886cb}{RK128}} whose cumulative proportional sum will have to be computed. \\
\hline
\mbox{\texttt{ in}}  & {\em max\+Array} & \+: The input scalar of the same type and kind as the input {\ttfamily Array} representing the maximum value in {\ttfamily Array} (i.\+e., {\ttfamily max\+Array = maxval(\+Array)}). \\
\hline
\mbox{\texttt{ in}}  & {\em backward} & \+: The input scalar of type {\ttfamily logical} of default kind. If {\ttfamily .true.}, the output {\ttfamily Cum\+Prop\+Exp} will be computed from the last element to the first element of the input {\ttfamily Array} ({\bfseries{optional}}, default = {\ttfamily .false.}). \\
\hline
\mbox{\texttt{ in}}  & {\em reversed} & \+: The input scalar of type {\ttfamily logical} of default kind. If {\ttfamily .true.}, the output array {\ttfamily Cum\+Prop\+Exp} will be reversed upon return, such that its last element becomes the first ({\bfseries{optional}}, default = {\ttfamily .false.}). \\
\hline
\mbox{\texttt{ in}}  & {\em cenabled} & \+: The input scalar of type {\ttfamily logical} of default kind. If {\ttfamily .true.}, then runtime checks for underflow occurrence will be enabled (hence the name {\ttfamily cenabled} standing for {\itshape Checks ENABLED}). Enabling this option can aid runtime efficiency when the division of a significant number of elements of {\ttfamily Array} (for example, half or more) by {\ttfamily max\+Array} causes underflow. In such cases, the exponentiation is avoided if {\ttfamily cenabled = .true.} leading to faster runtime by avoiding exponentiation since it is highly expensive (on the order of $\sim$200 CPU cycles). See \href{\#benchmark-genCumPropExp}{\texttt{ the relevant benchmark here}}. ({\bfseries{optional}}, default = {\ttfamily .false.}).\\
\hline
