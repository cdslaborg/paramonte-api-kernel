◆ getloglumdiswicmpc()

pure real(rk) function cosmology_mod::getloglumdiswicmpc ( real(rk), intent(in)  zplus1)

Return the approximate logarithm of the cosmological luminosity distance in units of MPc.

Parameters
[in]zplus1: The redshift plus 1.
Returns
logLumDisWicMpc : The approximate logarithm of the cosmological luminosity distance.
Warning
The approximation is accurate with a relative error of 0.001 for any redshift above 0.1, or zplus1 >= 1.1. Note that for redshifts less than 0.1, the error in the calculated luminosity distance grows to more than 0.001. This algorithm should therefore not be used for zplus1<0.1.
Remarks
The distance is calculated according to approximate algorithm of Wickramasinghe & Okwatta (2010).

real(rk) function cosmology_mod::getlookbacktime ( real(rk), intent(in)  zplus1,
real(rk), intent(in), optional  maxRelativeError,
integer(ik), intent(in), optional  nRefinement 
)

Return the cosmological lookback time in GYrs at the given redshift for the assumed cosmological parameters.

Parameters
[in]zplus1: The redshift plus 1.
[in]maxRelativeError: The maximum tolerance for error in the numerical integration (optional, default = 1.e-6).
[in]nRefinement: The number of refinements in the Romberg numerical integration (optional, default = 5).
Returns
lookBackTime : The cosmological lookback time in GYrs at the given redshift.
Remarks
The rest of the required parameters are taken from the module:
  • HUBBLE_TIME_GYRS : The Hubble time in units of Gyrs (Giga Years). It should be a number close to 13.8 Gyrs (Liddle, 2003, Page 57).
The integrations are performed using the Romberg integration method.

◆ ldiswickram()

pure real(rk) function cosmology_mod::ldiswickram ( real(rk), intent(in)  zplus1)

Return the approximation to the cosmological luminosity distance.

Parameters
[in]zplus1: The redshift plus 1.
Returns
ldiswickram : The approximate cosmological luminosity distance.

The approximation is accurate with a relative error of 0.001 for any redshift above 0.1.

Remarks
This function is the same as getLogLumDisWicMpc() function, except for the fact that it return the natural value, as opposed to the natural logarithm. It is kept only for legacy reasons and should not be used in new code.
