This is a method of the class ChainFileContents_type.
Return and store the contents of a ParaNest simulation output chain file in the object of class ChainFileContents_type.

Parameters
[in,out]CFC: The object of class ChainFileContents_type.
[in]chainFilePath: The list of variable names corresponding to each axis of the domain of the objective function.
[in]chainFileForm: The file format of the chain file ("binary" vs. "compact" vs. "verbose").
[out]Err: An object of class Err_type containing information about whether an error has occurred.
[in]lenHeader: The full length of the first line in the input file (the header line) (optional).
[in]ndim: The number of dimensions of the domain of the objective function (optional).
[in]delimiter: The delimiter symbol used in the chain file (optional, will be inferred if missing).
[in]chainSize: The size of the complete chain in the output file (optional, will be inferred if missing).
[in]allocSize: The allocation size of the chain (optional, default = chainSize).
Warning
Failure to provide the input optional argument chainSize will lead to the entire chain file being read into the chain object. This could potentially be catastrophic if the chain size is larger than the available virtual memory. At best, it will result in an IO twice slower than possible, since the file will have to be read twice.
When chainFileForm = "binary", lenHeader and delimiter must be also provided.
Remarks
If allocSize is missing in the input file, the entire file will be read into the chain object.
