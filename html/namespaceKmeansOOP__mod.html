Construct and return an object of KmeansOOP_type class. Note that this constructor solely allocates the components of the output Kmeans object to be later used for Kmeans clustering.

Parameters
[in]ndMax: The maximum possible number of dimensions (features) of the input data array Point.
[in]npMax: The maximum possible number of observations in the input data array Point.
[in]ncMax: The maximum possible number of clusters to be found via the Kmeans algorithm.
[in]ntMax: The maximum possible number of observations in the input data array Point.
[in]niterMax: The maximum number of iterations beyond which it a lack of converge is assumed (optional, default = 1000).
[in]nzsciMax: The maximum number of zero-sized cluster iterations beyond which it a lack of converge is assumed (optional, default = 10).
[in]relTol: The current requested relative tolerance below which should the potential fall, convergence is assumed (optional, default = 1.e-8).
Returns
Kmeans : An object of type KmeansOOP_type containing the properties of the identified clusters. The Errstat component of the output Kmeans will be set to 1 if the number of iterations reaches the input niterMax or its default value before convergence occurs. The Errstat component of the output Kmeans will be set to 2 if more than nzsciMax clustering tries result in clusters with zero members. In both cases in the above, Erroccurred = .true. upon exiting the procedure.
Warning
The input value for nt must be at least 1, otherwise it will lead to a runtime error or at best, the output will be undefined.
The input value for niterMax must be larger than 0 since at least one iteration will be performed by the procedure.
This algorithm does not check for the consistency of the input values for nc, nd, and np. It is currently the user's responsibility to ensure that these input values are logical and sensible.

This procedure is a method of the class KmeansOOP_type. Perform the Kmeans clustering on the input data set represented by the array Point(nd, np). Upon performing all iterations of clustering, the index Kmeansitopt will point to the final optimal iteration of Kmeans clustering which resulted in the smallest potential. If only one clustering try has been requested, then obviously, Kmeansitopt = 1.

Parameters
[in,out]Kmeans: An object of class KmeansOOP_type which will contain the clustering results.
[in]Point: The input array of size (nd,np) of the points to be clustered via the Kmeans algorithm.
[in]nd: The current number of data attributes to be used in clustering. The condition nd<=ndMax must hold at all times.
[in]np: The current number of input data points to be clustered. The condition np<=npMax must hold at all times.
[in]nc: The current number of Kmeans clusters to be requested. The condition nc<=ncMax must hold at all times.
[in]nt: The current number of Kmeans clustering trials. The condition nt<=ntMax must hold at all times.
[in]niterMax: The maximum number of iterations beyond which it a lack of converge is assumed (optional).
[in]nzsciMax: The maximum number of zero-sized cluster iterations beyond which it a lack of converge is assumed (optional).
[in]relTol: The current requested relative tolerance below which should the potential fall, convergence is assumed (optional).
[in]InitCenter: An input array of size (nd,nc,nt) representing the initial centers of the Kmeans clusters (optional). If not provided, the cluster centers will be initialized via the Kmeans++ algorithm.
Returns
Kmeans : An object of type KmeansOOP_type containing the properties of the identified clusters. The Errstat component of the output Kmeans will be set to 1 if the number of iterations reaches the input niterMax or its default value before convergence occurs. The Errstat component of the output Kmeans will be set to 2 if more than nzsciMax clustering tries result in clusters with zero members. In both cases in the above, Erroccurred = .true. upon exiting the procedure.
Warning
The input value for niterMax must be larger than 0 since at least one iteration will be performed by the procedure.
This algorithm does not check for the consistency of the input values for nc, nd, and np. It is currently the user's responsibility to ensure that these input values are logical and sensible.

Find nc initial cluster centers for the Kmeans algorithm, using Kmeans++ recipe.

Parameters
[in]nc: The number of clusters to be found via the Kmeans algorithm (must be > 1, though it is not verified).
[in]nd: The number of dimensions (features) of the input data array Point.
[in]np: The number of observations in the input data array Point.
[in]Point: The input array of size (nd,np) of the points to be clustered via the Kmeans algorithm.
[out]SumPoint: An output array of size (nd,nc) containing the sum of the points in each cluster. When divided by the corresponding output cluster size, it will yield the final updated cluster centers.
[out]Membership: An output array of size (np) representing the IDs of the clusters to which individual points belong.
[out]Size: An output array of size (nc) representing the sizes of the identified clusters.
[out]potential: The sum of the minimum-distances-squared of all points from their corresponding group centers.
Warning
There is no benefit in calling this routine for nc = 1. As such, calling this routine with nc = 1 will lead to undefined behavior.
