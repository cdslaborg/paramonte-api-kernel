Detailed Description

Return the sorted indices of the input contiguous Array of rank 1 in ascending order using the Quicksort algorithm, such that Array(Index) will be in ascending order.

Parameters
[in]Array: The contiguous array of rank 1 of either
  type real of kind RK32, RK64, or RK128 or,
  type integer of kind IK8, IK16, IK32, or IK64 or,
  type assumed-length character of default kind or,
  type String_type or also,
  a scalar assumed-length character of default kind,
  whose elements will be sorted in ascending order when called as Array(Index).
[out]Index: The output contiguous array of rank 1 of type integer of kind IK, IK32, or IK64 containing the sorted indices of Array such that Array(Index) is sorted in ascending order on return.
[in]lenArray: The input integer of kind IK32 or IK64 representing the length of the input array (optional. It must NOT be present if the length of Array is representable by the default integer kind IK and IK is neither IK32 nor IK64. Otherwise, it must be provided as an integer of kind IK32 if the array length is smaller than 2,147,483,647. Otherwise, it must be provided as an integer of kind IK64 if the array length is larger than 2,147,483,647.
Warning
If the length of the input Array is longer than the largest number that can be represented with an integer of kind IK32, then, the length of array must be provided as the input integer argument lenArray of kind IK64.
This procedure is impure when the library is compiled with macro CHECK_ENABLED=1. By default, it is pure in release build and impure in debug and testing builds.
Note
The procedures under this generic interface sort the input array by a mixture of Quicksort and straight insertion sorting methods. When the size the array to be sorted reaches 30 or less, the algorithm switches from Quicksort to straight insertion sorting.
In almost all scenarios, this algorithm must be the default choice for any array index sorting.
This algorithm is typically of order \( N \log_2 N\), and the worst-case order of \( N^2\). The worst case performance occurs for completely sorted input arrays.
To sort the input array in descending order, simply call the output Index elements from the end to the beginning or, rewrite the array with Array = Array( Index( ubound(Array, dim = 1) : lbound(Array, dim = 1) : -1 ) ).
A rooter array can be sorted along with a leader array with the help of sortIndex.
