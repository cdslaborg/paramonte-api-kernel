92  chainfileformatobj%isVerbose = .false.
93  chainfileformatobj%isBinary = .false.
94  chainfileformatobj%compact = "compact"
95  chainfileformatobj%verbose = "verbose"
96  chainfileformatobj%binary = file_type%binary
97  chainfileformatobj%def = chainfileformatobj%compact
98 
99  chainfileformatobj%null = repeat(null_sk, max_len_chain_file_format)
100 
101  chainfileformatobj%desc = &
102  "chainFileFormat is a string variable that represents the format of the output chain file(s) of "// methodname // &
103  " simulation. The string value must be enclosed by either single or double quotation marks when provided as input. &
104  &Three values are possible:\n\n&
105  & chainFileFormat = 'compact'\n\n&
106  & This is the ASCII (text) file format which is human-readable but does not preserve the full accuracy of the &
107  &output values. It is also a significantly slower mode of chain file generation, compared to the binary file format (see below). &
108  &If the compact format is specified, each of the repeating MCMC states will be condensed into a single entry (row) in &
109  &the output MCMC chain file. Each entry will be then assigned a sample-weight that is equal to the number of repetitions of &
110  &that state in the MCMC chain. Thus, each row in the output chain file will represent a unique sample from the objective function. &
111  &This will lead to a significantly smaller ASCII chain file and faster output size compared to the verbose chain file format (see below).\n\n&
112  & chainFileFormat = 'verbose'\n\n&
113  & This is the ASCII (text) file format which is human-readable but does not preserve the full accuracy of &
114  &the output values. It is also a significantly slower mode of chain file generation, &
115  &compared to both compact and binary chain file formats (see above and below). &
116  &If the verbose format is specified, all MCMC states will have equal sample-weights of 1 in the output chain file. &
117  &The verbose format can lead to much larger chain file sizes than the compact and binary file formats. &
118  &This is especially true if the target objective function has a very high-dimensional state space.\n\n&
119  & chainFileFormat = '" // chainfileformatobj%binary // "'\n\n&
120  & This is the binary file format which is not human-readable, but preserves the exact values in the output &
121  &MCMC chain file. It is also often the fastest mode of chain file generation. If the binary file format is chosen, the chain &
122  &will be automatically output in the compact format (but as binary) to ensure the production of the smallest-possible output chain file. &
123  &Binary chain files will have the " // file_ext%binary // " file extensions. Use the binary format if you need full accuracy representation &
124  &of the output values while having the smallest-size output chain file in the shortest time possible.\n\n&
125  &The default value is chainFileFormat = '" // chainfileformatobj%def // "' as it provides a reasonable trade-off between &
126  &speed and output file size while generating human-readable chain file contents. Note that the input values are case-insensitive."
127  end function constructchainfileformat
