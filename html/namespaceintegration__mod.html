Return the integral of function getFunc in the closed range [lowerLim,upperLim] using Adaptive Romberg extrapolation method.

Parameters
[in]getFunc: The input function to be integrated. It must have the interface specified by integrand_proc.
[in]lowerLim: The lower limit of integration.
[in]upperLim: The upper limit of integration.
[in]maxRelativeError: The tolerance that once reach, the integration will stop.
[in]nRefinement: The number of refinements in the Romberg method. A number between 4-6 is typically good.
[out]integral: The result of integration.
[out]relativeError: The final estimated relative error in the result. By definition, this is always smaller than maxRelativeError.
[out]numFuncEval: The number of function evaluations made.
[out]ierr: Integer flag indicating whether an error has occurred. If ierr == 0, then no error has occurred. Otherwise, the integer value points to the corresponding element of ErrorMessage array.

Return the refinement of the integration of an exponentially-decaying function on a semi-infinite. This function is an integrator driver to be passed to doQuadRombOpen.

Parameters
[in]getFunc: The input function to be integrated. It must have the interface specified by integrand_proc.
[in]lowerLim: The lower limit of integration.
[in]upperLim: The upper limit of integration (typically set to huge(1._RK) to represent \(+\infty\)).
[in,out]integral: The result of integration.
[in]refinementStage: The number of refinements since the first call to the integrator.
[out]numFuncEval: The number of function evaluations made.
Remarks
It is expected that upperLim > lowerLim > 0.0 must hold for this integrator to function properly.

subroutine integration_mod::midinf ( procedure(integrand_proc)  getFunc,

)

This routine is an exact replacement for midpnt, i.e., returns as integral the nth stage of refinement of the integral of funk from lowerLim to upperLim, except that the function is evaluated at evenly spaced points in 1/x rather than in x. This allows the upper limit upperLim to be as large and positive as the computer allows, or the lower limit lowerLim to be as large and negative, but not both.

Parameters
[in]getFunc: The input function to be integrated. It must have the interface specified by integrand_proc.
[in]lowerLim: The lower limit of integration.
[in]upperLim: The upper limit of integration.
[in,out]integral: The result of integration.
[in]refinementStage: The number of refinements since the first call to the integrator.
[out]numFuncEval: The number of function evaluations made.
Warning
lowerLim * upperLim > 0.0 must hold for this integrator to function properly.

subroutine integration_mod::midpnt ( procedure(integrand_proc)  getFunc,
real(rk), intent(in)  lowerLim,
real(rk), intent(in)  upperLim,
real(rk), intent(inout)  integral,
integer(ik), intent(in)  refinementStage,
integer(ik), intent(out)  numFuncEval 
)

This routine computes the nth stage of refinement of an extended midpoint rule. When called with n = 1, the routine returns as integral the crudest estimate of \(\int_a^b f(x) ~ dx\). Subsequent calls with n = 2, 3, ... (in that sequential order) will improve the accuracy of integral by adding (2/3) * 3n-1 additional interior points.

Parameters
[in]getFunc: The input function to be integrated. It must have the interface specified by integrand_proc.
[in]lowerLim: The lower limit of integration.
[in]upperLim: The upper limit of integration.
[in,out]integral: The result of integration.
[in]refinementStage: The number of refinements since the first call to the integrator.
[out]numFuncEval: The number of function evaluations made.
Warning
The argument integral should not be modified between sequential calls.
