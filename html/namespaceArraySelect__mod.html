Detailed Description

This module contains procedures for removing a selection from arrays of various types at the specified instances of occurrence of selection.

Visualization of the benchmark output ⛓

Benchmark moral ⛓
  1. The procedures under the generic interface select take both scalar and vector Selection arguments. As evidenced by the above benchmark, when the input Selection is vector of length 1, it is much faster, by 4X or more, to pass Selection as a scalar instead of a whole array of length 1. This benchmark represents the worst-case scenario. Note that this benchmark is not relevant to selecting substrings to Fortran strings.

Postprocessing of the output ⛓

Visualization of the benchmark output ⛓

Benchmark moral ⛓
  1. The procedures under the generic interface genSelected are functions while the procedures under the generic interface select are subroutines. From the benchmark results, it appears that the functional interface performs slightly less efficiently than the subroutine interface. The sole apparent reason for this performance loss seems to be the extra copy of the result to the allocatable ArrayNew on return from the function. Note that this benchmark does not even include the cost of repeated reallcations, that is, the allocation of ArrayNew happen only once in all tests.
  2. Note that this benchmark considers the worst-case scenario where Selection must be selected at all positions of the input Array.
