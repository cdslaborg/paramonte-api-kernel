97  restartfileformatobj%desc = &
98  "restartFileFormat is a string variable that represents the format of the output restart file(s) which are used to restart &
99  &an interrupted "// methodname //" simulation. The string value must be enclosed by either single or double quotation &
100  &marks when provided as input. Two values are possible:\n\n&
101  & restartFileFormat = '" // restartfileformatobj%binary // "'\n\n&
102  & This is the binary file format which is not human-readable, but preserves the exact values of the &
103  &specification variables required for the simulation restart. This full accuracy representation is required &
104  &to exactly reproduce an interrupted simulation. The binary format is also normally the fastest mode of restart file &
105  &generation. Binary restart files will have the " // file_ext%binary // " file extensions.\n\n&
106  & restartFileFormat = '" // restartfileformatobj%ascii // "'\n\n&
107  & This is the ASCII (text) file format which is human-readable but does not preserve the full accuracy of &
108  &the specification variables required for the simulation restart. It is also a significantly slower mode of &
109  &restart file generation, compared to the binary format. Therefore, its usage should be limited to situations where &
110  &the user wants to track the dynamics of simulation specifications throughout the simulation time. &
111  &ASCII restart file(s) will have the " // file_ext%ascii //" file extensions.\n\n&
112  &The default value is restartFileFormat = '" // restartfileformatobj%def // "'. Note that the input values are case-insensitive."
113  end function constructrestartfileformat
