Detailed Description

Solve the set of nd linear equations of the form,.

\begin{equation} A \cdot X = B \end{equation}

where \(A\) is the coefficient matrix represented by its Cholesky lower-triangular decomposition matrix CoefMatChoLow and its diagonals CoefMatChoDia, \(X\) is the output solution, and \(B\) is the right-hand-side vector of constants.

Parameters
[in,out]Solution: The output contiguous vector of size nd of type real of kind RK32, RK64, or RK128, containing the solution \(X\). If the input argument RightHand is missing, then Solution also represents the right-hand-side \(B\) in the system of linear equations to be solved on input. On output, it will be completely overwritten with the solution.
[in]CoefMatChoLow: The input contiguous square matrix of shape (nd,nd) of the same type and kind as Solution, representing the Cholesky lower-triangular decomposition of the matrix of coefficients \(A\) of the linear system. This matrix is returned by the call to getChoLow.
[out]CoefMatChoDia: The input contiguous array of shape (nd) of the same type and kind as Solution, representing the diagonal elements of the Cholesky factorization of the matrix of coefficients \(A\) of the linear system. This vector is returned by the call to getChoLow.
[in]RightHand: The input contiguous vector of size nd of the same type and kind as Solution, representing the right-hand-side \(B\) in the system of linear equations to be solved (optional, if missing, then \(B\) must be provided as input in Solution).

Warning
The dimensions of the input arguments Solution, CoefMatChoLow, and CoefMatChoDia must be consistent with each other as described in the above. This condition is only checked for when the preprocessor macro CHECK_ENABLED=1 is set at build time.
This pure procedures documented herein become impure when the ParaMonte library is compiled with preprocessor macro CHECK_ENABLED=1. By default, pure procedures are pure in release build and impure in debug and testing builds.
Note
The input CoefMatChoLow and CoefMatChoDia arguments are not modified by this routine and, can be left in place for successive calls with different right-hand sides Solution.
