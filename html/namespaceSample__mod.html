◆ genInvCovMat()

real(RK) function, dimension(:,:), allocatable Sample_mod::genInvCovMat ( real(RK)  ShiftedSample,
real(RK)  Weight,
logical, intent(out), optional  failed 
)

Return the full inverse covariance matrix of the input ShiftedSample sample (whose mean is shifted to the origin) whose shape is assumed to be (nd,np) or (nd,np) such that nd < np always holds, where nd represents the number of attributes (dimensions) of the sample and np represents the number of data points in the sample. Note that by definition, np > nd must hold. The input sample must be normalized in the sense that its mean is zero.

Parameters
[in]ShiftedSample: The contiguous array of shape (:,:) containing the sample.
[in]Weight: The contiguous array of shape (:,:) containing the sample.
[out]failed: An output logical that is .true., it implies that the Cholesky factorization has failed. If missing among the dummy arguments, the procedure will end the program upon encountering a Cholesky factorization failure.
Returns
InvCovMat : The full inverse covariance matrix of shape (nd,nd) of the input normalized sample.
Note
If needed, the sample normalization before passing it to this procedure can be done by calling the appropriate function under the generic interface Sample_mod::genShifted().

Return the lower triangle Cholesky Factor of the covariance matrix of a set of points in the lower part of ChoLowCovUpp. The upper part of ChoLowCovUpp, including the diagonal elements of it, will contain the covariance matrix of the sample. The output argument ChoDia, contains the diagonal terms of Cholesky Factor.

Parameters
[in]nd: The number of dimensions of the input sample.
[in]np: The number of points in the sample.
[in]Mean: The mean vector of the sample.
[in]Sample: The array of shape (nd,np) containing the sample.
[out]ChoLowCovUpp: The output matrix of shape (nd,nd) whose lower triangle contains elements of the Cholesky factor. The upper triangle of the matrix contains the covariance matrix of the sample.
[out]ChoDia: The diagonal elements of the Cholesky factor.
Warning
If Cholesky factorization fails, ChoDia(1) will be set to -1 to indicate an error on return. This is necessary to maintain the purity of the procedure. However, if the library is built with the preprocessor macro CHECK_ENABLED set to true, then the occurrence of an error will halt the program by executing the error stop statement.
This procedure is impure when the library is compiled with macro CHECK_ENABLED=1. By default, it is pure in release build and impure in debug and testing builds.

Return the lower triangle Cholesky Factor of the covariance matrix of a set of points in the lower part of ChoLowCovUpp. The upper part of ChoLowCovUpp, including the diagonal elements of it, will contain the covariance matrix of the sample. The output argument ChoDia, contains the diagonal terms of Cholesky Factor. This routine delivers the same functionality of getChoLowCovUpp(). However, it is considerably faster, by a few factors, for nd >> 10. Furthermore, it requires significantly less runtime memory, by about half. The exact amount of speedup depends heavily on the architecture and memory layout of the runtime system.

Parameters

[in]Mean: The mean vector of the sample.
[in]Sample: The array of shape (nd,np) containing the sample.
[out]ChoLowCovUpp: The output matrix of shape (nd,nd) whose lower triangle contains elements of the Cholesky factor. The upper triangle of the matrix contains the covariance matrix of the sample.
[out]ChoDia: The diagonal elements of the Cholesky factor.

Return the transformed sample according to the input Cholesky factorization.

Parameters
[in]Sample: The contiguous array of data points of shape (:,:) normalized to their mean vector. By default, the shape of Sample is assumed to be (nd,np) unless dim = 1.
[in]ChoLow: The input lower-triangle Cholesky factorization of shape (nd,nd) with which the input sample must be transformed.
[in]ChoDia: The input diagonal elements of the Cholesky factorization of the transformation. (optional, if not provided, the procedure assumes that the diagonal elements are given in ChoLow)
[in]dim: The dimension of the input Sample along which individual data exist (optional, default = 2_IK).
[in]tenabled: A logical input. If .true., the output transformed sample will have the transposed shape of the input Sample. (optional, default = .false.).
Returns
TranSample : The output transformed sample based on the input Cholesky factorization.
