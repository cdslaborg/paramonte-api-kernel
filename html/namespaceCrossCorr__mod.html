Compute the autocorrelation of the input data matrix (that is already normalized with respect to its mean).

Parameters
[in]nd: The number of dimensions of the input data (the number of data features).
[in]np: The length of the input data (the number of observations or points).
[in]NormedData: The input data of size (nd,np). On input, it must be normalized with respect to it mean vector.
[in]nlag: The length of the input vector of lags.
[in]Lag: The input vector of lags at which the autocorrelation must be computed.
[out]AutoCorr: The output AutoCorrelation matrix of shape (nd,nlag).
[in]InverseSumNormedDataSq: The inverse sum of the square of the input normalized data (optional).
Remarks
If InverseSumNormedDataSq is provided, the computations will be slightly faster.
This function uses the direct method of covariance computation for computing the autocorrelation. It is therefore highly inefficient and not recommended for use in HPC solutions. Use instead, getCrossCorrFFT or getCrossCorrWeightedFFT.

◆ getCrossCorrFFT()

real(RK) function, dimension(paddedLen) CrossCorr_mod::getCrossCorrFFT ( integer(IK)  paddedLen,
real(RK)  PaddedData1,
real(RK)  PaddedData2 
)

Return the cross-correlation of the two input data vectors, (including any user-supplied zero padding), computed via Fast-Fourier Transform.

Parameters
[in]paddedLen: The lengths of the input arrays, which MUST be an integer power of two.
[in]PaddedData1: The first array, possibly padded with zero to become an array of length paddedLen.
[in]PaddedData2: The second array, possibly padded with zero to become an array of length paddedLen.
Returns
CrossCorrFFT : A vector of same length as the input arrays containing the cross-correlation. The answer is returned as the first paddedLen points in ans stored in wrap-around order, i.e., cross-correlation at increasingly negative lags are in CrossCorrFFT(paddedLen) on down to CrossCorrFFT(paddedLen/2+1), while cross-correlation increasingly positive lags are in CrossCorrFFT(1) (zero lag) on up to CrossCorrFFT(paddedLen/2).
Remarks
Sign convention of this routine: If PaddedData1 lags PaddedData2, i.e., is shifted to the right of it, then ans will show a peak at positive lags.
For autocorrelation, under the assumption of a completely random series, the ACF standard error reduces to: \(\sqrt{ 1 / \texttt{paddedLen} }\)

[in]CompactData1: The first array, in compact weighted format.

Returns

integer(IK), intent(in), optional  base 
)

Return the smallest length of a vector that is a power of base and at least base**2 times larger than the input length actualLen.

Parameters
[in]actualLen: The input vector length.
[in]base: The integer-valued base of the exponentiation (optional, default = 2).
Returns
paddedLen : The minimum power-of-base length given actualLen.
Remarks
This method is used to compute the cross-correlation. Usage: actualLen = max( actualLen1, actualLen2 )
Warning
The input values for absoluteValue and base must be both larger than 1.
Remarks
For weighted-data cross-correlation computation, try, actualLen = max( sum(Weight1(1:actualLen1)), sum(Weight2(1:actualLen2)) ).
