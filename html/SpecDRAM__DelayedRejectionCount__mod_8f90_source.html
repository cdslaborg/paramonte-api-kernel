87  self%desc = &
88  num2str(min_delayed_rejection_count) // " <= delayedRejectionCount <= " // num2str(max_delayed_rejection_count) // &
89  " is an integer that represents the total number of stages for which rejections of new proposals &
90  &will be tolerated by "//methodname//" before going back to the previously accepted point (state). &
91  &Possible values are:\n\n&
92  & delayedRejectionCount = 0\n\n&
93  & indicating no deployment of the delayed rejection algorithm.\n\n&
94  & delayedRejectionCount > 0\n\n&
95  & which implies a maximum delayedRejectionCount number of rejections will be tolerated.\n\n&
96  &For example, delayedRejectionCount = 1, means that at any point during the sampling, if a proposal is rejected, "&
97  //methodname//" will not go back to the last sampled state. Instead, it will continue to propose a new from the current &
98  &rejected state. If the new state is again rejected based on the rules of "//methodname//", then the algorithm will not &
99  &tolerate further rejections, because the maximum number of rejections to be tolerated has been set by the user to be &
100  &delayedRejectionCount = 1. The algorithm then goes back to the original last-accepted state and will begin proposing &
101  &new states from that location. &
102  &The default value is delayedRejectionCount = "// num2str(self%def) // "."
103  end function constructdelayedrejectioncount
