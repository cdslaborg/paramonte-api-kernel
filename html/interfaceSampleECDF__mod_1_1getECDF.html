SampleECDF_mod::getECDF Interface Reference

Compute and return the Empirical Cumulative Distribution Function (ECDF) of the input univariate sample. More...

Detailed Description

Compute and return the Empirical Cumulative Distribution Function (ECDF) of the input univariate sample.

Parameters
[in,out]Sample: The contiguous array of shape (np) (rank 1) of either
  type real of kind RK32, RK64, or RK128 or
  type integer of kind IK8, IK16, IK32, or IK64
  containing the univariate sample, where np is the number of data points in the sample.
  On return, the input Sample will be sorted in ascending order.
[out]ECDF: The output contiguous array of rank 1 of the same size as the input Sample containing the Empirical Cumulative Distribution Function (ECDF) of the input univariate Sample. If the input Sample is of type real of kind RK32, RK64, or RK128, then ECDF must be the same type and kind as Sample. If the input Sample is of type integer of kind IK8, IK16, IK32, or IK64, then ECDF must be of type real of kind RK32, RK64, or RK128.
[out]Lower: The output contiguous array of the same type, kind, rank, and size as the output ECDF containing the lower confidence bound on the ECDF as the specified \(1-\alpha\) probability (optional, if missing, it is not computed).
[out]Upper: The output contiguous array of the same type, kind, rank, and size as the output ECDF containing the upper confidence bound on the ECDF as the specified \(1-\alpha\) probability (optional, if missing, it is not computed).
[in]alpha: The input scalar of type real of the same kind as ECDF from which the probability \(1-\alpha\) for the upper and lower confidence bounds of ECDF is computed. For example, if \(\alpha = 0.05\), then the output Lower and Upper confidence bounds contain the CDF of the true parent distribution of the sample with \( 95\% \) probability. In such a case, the lower bound marks the \( \alpha / 2 \) probability contour and the upper bound marks the \( 1 - \alpha / 2 \) probability contour (optional, default = 0.05).
[in]sorted: The input logical of default kind. If .true., the sample is assumed to be sorted on input and will not be sorted. If the input Sample is already sorted, set sorted = .true. as it will lead to much faster ECDF computations (optional, default = .false., that is, the sample will be sorted by the procedure prior to computing the ECDF).
Warning
This procedure is impure when the library is compiled with macro CHECK_ENABLED=1. By default, it is pure in release build and impure in debug and testing builds.
