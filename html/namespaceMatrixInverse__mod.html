pure real(RK) function, dimension(nd,nd) MatrixInverse_mod::genInvChoLow ( integer(IK), intent(in)  nd,
real(RK)  MatLow,
real(RK)  Diagonal 
)

Return the inverse matrix InvMatLow of an input lower-triangle matrix of shape (nd,nd) whose diagonal elements are given separately as the input Diagonal of length nd. The upper triangle of the input matrix will not be used.

Parameters
[in]nd: The size of the input matrix.
[in,out]MatLow: The lower triangle input matrix (except the diagonal terms which are provided separately).
[in,out]Diagonal: The diagonal elements of the lower triangle matrix MatLow.
Returns
InvMatLow : The lower-triangle inverse matrix of the input lower-triangle matrix.
Warning
The output is not a full inverse matrix. In particular, the upper-triangle elements of the output matrix are not set to zero.
Remarks
This procedure is particularly useful for computing the inverse of the lower-triangle of Cholesky factorization returned by the procedure getChoLow.

pure real(RK) function, dimension(nd,nd) MatrixInverse_mod::genInvLowFromChoLow ( integer(IK), intent(in)  nd,
real(RK)  ChoLow,
real(RK)  ChoDia 
)

Generate the lower-triangle and diagonals of the inverse matrix of a symmetric-positive-definite matrix whose Cholesky Lower triangle is given in the lower part of ChoLow and and its diagonal elements in ChoDia.

Parameters
[in]nd: The size of the input square matrix - nd by nd.
[in]ChoLow: An array of shape (nd,nd) whose lower triangle contains the lower triangle of the Cholesky factorization of the matrix. The upper triangle is not needed.
[in]ChoDia: A vector of length nd containing the diagonal elements of the Cholesky factorization of the matrix.
Returns
InvLowFromChoLow : The full symmetric inverse matrix.
Remarks
While there is not harm in using this routine for the case when nd = 1, it is redundant. Note that InvLowFromChoLow = 1._RK / ChoDia(1)^2 when nd = 1.

real(RK) function, dimension(nd,nd) MatrixInverse_mod::genInvMat ( integer(IK), intent(in)  nd,
real(RK)  Matrix,
logical, intent(out), optional  failed 
)

Return the inverse matrix InverseMatrix of a (nd,nd) input matrix Matrix, and its determinant, using LU decomposition.

Parameters
[in]nd: The size of the square matrix and its LU decomposition - nd by nd.
[in]Matrix: The target symmetric-positive-definite matrix.
[out]failed: The output logical that is .true. only if the LU-factorization fails, indicating the occurrence of an error. (optional, if missing, the program will halt by executing error stop upon occurrence of an error.)
Returns
InverseMatrix : The full inverse matrix.
Remarks
This routine has the same functionality as the subroutine getInvMatDet, but returns the result as a function output. According to the timing tests, compare_InvMatRoutines_1(), this function version appears to be 15-30% faster than the subroutine equivalent getInvMatDet.

Generate the inverse matrix of an input symmetric-positive-definite matrix PosDefMat.

Parameters
[in]nd: The size of the input square matrix - nd by nd.
[in]PosDefMat: The input symmetric-positive-definite matrix.
Returns
InvPosDefMat : The full inverse matrix.
Warning
If the input matrix is not positive-definite and the library is built with the preprocessor macro CHECK_ENABLED set to true, then the program will halt by executing the error stop statement. If the preprocessor macro CHECK_ENABLED is undefined or false, the function will return with InvPosDefMat(1,1) = -1._RK to indicate the occurrence of an error.
This pure procedures documented herein become impure when the ParaMonte library is compiled with preprocessor macro CHECK_ENABLED=1. By default, pure procedures are pure in release build and impure in debug and testing builds.
Do not call this routine when nd = 1. For nd = 1: InvPosDefMat = 1._RK / InvPosDefMat.
Remarks
On input, only the upper half of PosDefMat needs to be given.
This routine has the same functionality as the subroutine getInvPosDefMatSqrtDet, but returns the result as a function output. According to the timing tests, compare_InvMatRoutines_1(), the function version appears to be 15-30% faster than the subroutine version getInvPosDefMatSqrtDet.

[out]failed: The output logical that is .true. only if the LU-factorization fails, indicating the occurrence of an error.

Remarks
This routine has the same functionality as the function genInvMat, but returns the result as subroutine arguments. According to the timing tests, compare_InvMatRoutines_1(), the function version of this code genInvMat appears to be 15-30% faster than this subroutine version.

PURE subroutine MatrixInverse_mod::getInvPosDefMatSqrtDet ( integer(IK), intent(in)  nd,
real(RK)  MatInvMat,
real(RK)  sqrtDetInvPosDefMat 
)

Return the inverse matrix of a symmetric-positive-definite input matrix, which is given in the upper triangle of MatInvMat. On output MatInvMat is completely overwritten by in the inverse of the matrix. Also, return the square root of determinant of the inverse matrix.

Parameters
[in]nd: The size of the input square matrix - nd by nd.
[in,out]MatInvMat: The input square matrix. On input, the upper half must be covariance matrix. On output, it is completely overwritten by the inverse matrix.
[out]sqrtDetInvPosDefMat: The square root of the determinant of the inverse matrix.
Warning
Do not call this routine when nd = 1. There is no need to call this routine when nd = 1.
If the input matrix is not positive-definite and the library is built with the preprocessor macro CHECK_ENABLED set to true, then the program will halt by executing the error stop statement. If the preprocessor macro CHECK_ENABLED is undefined or false, then the output sqrtDetInvPosDefMat will be set to -1 on return to indicate the occurrence of an error.
This pure procedures documented herein become impure when the ParaMonte library is compiled with preprocessor macro CHECK_ENABLED=1. By default, pure procedures are pure in release build and impure in debug and testing builds.
