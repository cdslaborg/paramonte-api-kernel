Detailed Description

Generate a diagonal matrix of shape (nrow,ncol) with its diagonal elements set to the input Diag. All off-diagonal elements are set to zero.

Parameters
[in]Diag: The input scalar or contiguous array of rank 1 of either
  type real of kind RK32, RK64, or RK128 or
  type complex of kind CK32, CK64, or CK128
  containing the diagonal elements of the output matrix.
[in]nrow: The input positive integer of default kind IK representing the number of rows of the output matrix. (optional, if and only if Diag is array-like, in which case, default = size(Diag) + srow - 1).
[in]ncol: The input positive integer of default kind IK representing the number of columns of the output matrix. (optional, if and only if Diag is array-like, in which case, default = size(Diag) + scol - 1).
[in]srow: The input positive integer of default kind IK representing the starting row to write the input Diag elements to the output matrix DiagMat (optional, default = 1).
[in]scol: The input positive integer of default kind IK representing the starting column to write the input Diag elements to the output matrix DiagMat (optional, default = 1).
[in]offdiag: The input scalar of the type and kind as the input Diag to be used as the value for all off-diagonal elements of the output diagonal matrix.
Returns
DiagMat : The output diagonal matrix of the same type and kind as the input Diag and shape (nrow,ncol) (if nrow and ncol are present) or (size(Diag) + srow - 1, size(Diag) + scol - 1) (if nrow and ncol are missing as input arguments. The diagonal elements of DiagMat are set to the input Diag values. All other matrix elements will be set to offdiag.

Warning
If the input Diag is a contiguous array and nrow and ncol are also present as input arguments, then the conditions nrow >= srow + size(Diag) - 1 and ncol >= scol + size(Diag) - 1 must hold. These conditions are checked only when the preprocessor flag CHECK_ENABLED=1 is set at build time.
This pure procedures documented herein become impure when the ParaMonte library is compiled with preprocessor macro CHECK_ENABLED=1. By default, pure procedures are pure in release build and impure in debug and testing builds.
