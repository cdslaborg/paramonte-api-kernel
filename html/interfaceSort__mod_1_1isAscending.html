Sort_mod::isAscending Interface Reference

Generate and return .true. if the input array is sorted in ascending order (with the possibility elements being equal), otherwise, generate and return .false.. More...

Detailed Description

Generate and return .true. if the input array is sorted in ascending order (with the possibility elements being equal), otherwise, generate and return .false..

Parameters
[in]Array: The contiguous array of rank 1 of either
  type real of kind RK32, RK64, or RK128 or,
  type integer of kind IK8, IK16, IK32, or IK64 or,
  type assumed-length character of default kind or,
  type String_type or,
  also, a scalar assumed-length character of default kind,
  whose elements will be checked for an all ascending order (with the possibility elements being equal).
Returns
sorted : An output logical of default kind that is .true. if all elements of the input array are in ascending order (with the possibility elements being equal) or all equal. Otherwise, it is .false..
Warning
The output of this procedure is .true. when the input array has zero length.
This procedure is impure when the library is compiled with macro CHECK_ENABLED=1. By default, it is pure in release build and impure in debug and testing builds.
