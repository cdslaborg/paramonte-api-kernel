Detailed Description

Generate and return the natural logarithm of the sum of the exponential of the input Array robustly (without numerical overflow).

Parameters
[in]Array: The input contiguous array of shape (:) of either
&bsp; type real of kind RK32, RK64, or RK128 or,
&bsp; type complex of kind CK32, CK64, or CK128,
whose natural logarithm of sum of the exponential of its elements will be generated and returned.
[in]maxArray: The input scalar of the same type and kind as the input Array representing the maximum value in Array (i.e., maxArray = maxval(Array)). If the input Array is of complex type.
[in]cenabled: The input scalar of type logical of default kind. If .true., then runtime checks for underflow occurrence will be enabled (hence the name cenabled standing for Checks ENABLED). Enabling this option can aid runtime efficiency when the division of a significant number of elements of Array (for example, half or more) by maxArray causes underflow. In such cases, the exponentiation is avoided if cenabled = .true. leading to faster runtime by avoiding exponentiation since it is highly expensive (on the order of ~200 CPU cycles). See the relevant benchmark here. (optional, default = .false.).
Returns
logSumExp : The output scalar of the same type and kind as the input Array containing the natural logarithm of the sum of the exponential of the input Array robustly (without numerical overflow).

Benchmark moral ⛓
  1. If the input array has many (half the size of array or more) elements whose division by the maxval(Array) causes underflow, then setting cenabled = .true. when calling genLogSumExp will likely result in a faster runtime. Conversely, if the divisions are not expected to cause any or too many underflows, then set cenabled = .true. to improve cache coherence and runtime performance (at the expense of occasional expensive but redundant exponentiations).
  2. Note the performance of manual computation against genLogSumExp with or without setting the optional argument cenabled.
