String_mod::val2char Interface Reference

Detailed Description

Convert the input value to an output allocatable character, with the requested format and sign symbol, if provided.

Parameters
[in]val: The input scalar or contiguous array of rank 1 or 2 of either
  type complex of kind CK32, CK64, or CK128 or   type real of kind RK32, RK64, or RK128 or   type integer of kind IK8, IK16, IK32, or IK64 or   type logical of default kind.
[in]format: The Fortran IO format to be used when writing the value to the output allocatable character (optional, the default value depends on the type of the input val:
  If val is of type complex and the sign argument is missing, then default ‘format = "(*(&rsquo;(',g0,',',g0,')',:,','))"`.
  If `val` is of type `complex` and `sign = .true.`, then default `format = "(*('(',sp,g0,',',g0,')',:,','))"`.
  If `val` is of type `real` and the `sign` argument is missing, then default `format = "(*(g0,:,','))"`.
  If `val` is of type `real` and the `sign` argument is missing, then default `format = "(*(sp,g0,:,','))"`.
).
[in]len: The length of the output string. If provided, the output string will be preallocated with the requested input length. Then, the value will be written to it. If missing, the blank characters on both ends of the output string will be trimmed. (optional, default = automatically determined such that all leading and trailing blank characters are removed)
Returns
string : The output string.
Remarks
This procedure is a static method of the class String_type.
Warning
Note that len, if provided, must be large enough to contain the full input value.
