Detailed Description

Sort the input contiguous Array of rank 1 in ascending order, using a mixture of insertion-sort and merge-sort algorithms.

Parameters
[in,out]Array: The contiguous array of rank 1 of either type real of kind RK32, RK64, or RK128 or type integer of kind IK8, IK16, IK32, or IK64 whose elements will be sorted in ascending order on return.

Warning
The size of the input array must not exceed the maximum value representable by the default integer kind IK.
This algorithm is provided only for comparison and benchmarking purposes. Its usage for generic applications is not recommended. The sort algorithm should be the default choice for any sorting in almost all scenarios.
This algorithm is recursive and as such, requires stack allocation, which could lead to runtime stack-overflow errors.
This pure procedures documented herein become impure when the ParaMonte library is compiled with preprocessor macro CHECK_ENABLED=1. By default, pure procedures are pure in release build and impure in debug and testing builds.
Note
This algorithm is typically of order \( N \log_2 N\).
To sort the input array in descending order, simply call the output array elements from the end to the beginning or, rewrite the array with Array = Array( ubound(Array, dim = 1) : lbound(Array, dim = 1) : -1 ).
A rooter array can be sorted along with a leader array with the help of sortIndex.
