Detailed Description

This module contains types and procedures related to the following:

  1. the construction of ellipsoids from a given set of points.
  2. the computation of the volume and surface area of hyper-ellipsoids.

Return the (bounding) ellipsoid of the input set of Points.

Parameters
[in]nd: The number of dimensions.
[in]np: The number of input points.
[in]Point: The input array of points of shape (nd,np).
[in]isBounding: A logical value that if .true. will yield the bounding ellipsoid of the input Point dataset.
Returns
Ellipsoid : An object of type Ellipsoid_type.
Warning
If any error occurs during the ellipsoid construction, the first element of the ChoDia component of the output Ellipsoid object will be set to a negative number on return. This is to preserve the purity of the function and to avoid the need for error handling.
Remarks
When isBounding = .true., the scaleFactor, scaleFactorSq, logVolNormed are computed as scaled.

Return a random point on the target ellipsoid by projecting a random point uniformly distributed within the ellipsoid on its boundary.

Parameters
[in]nd: The number of dimensions of the ellipsoid.
[in]Mean: The mean vector (center) of the ellipsoid.
[in]ChoLow: The Cholesky lower triangle of the representative matrix of the ellipsoid.
[in]ChoDia: The diagonal elements of the Cholesky lower triangle of the representative matrix of the ellipsoid.
Returns
RandPointOnEllipsoid : A random point on the target ellipsoid by projecting a random point uniformly distributed within the ellipsoid on its boundary.
Remarks
This is algorithm is similar to DomainEllipsoid_mod::getRandUnif, with the only difference that points are drawn randomly from the surface of the ellipsoid instead of inside of its interior.
Note that the distribution of points on the surface of the ellipsoid is NOT uniform. Regions of high curvature will have more points randomly sampled from them. Generating uniform random points on arbitrary-dimension ellipsoids is not a task with trivial solution!
Todo:
The performance of this algorithm can be further improved.

◆ getLogVolChoLowEllipsoidFromRepMatUpp()

real(RK) function GeometryEllipsoid_mod::getLogVolChoLowEllipsoidFromRepMatUpp ( real(RK)  ChoLowRepMatUpp,
real(RK)  ChoDia,
logical, intent(out), optional  failed,
real(RK)  logVolUnitBall 
)

Return the volume of a hyper-ellipsoid represented by the input upper-triangle representative matrix.

Parameters
[in,out]ChoLowRepMatUpp: The upper-triangle of the square representative matrix of shape (nd,nd) of the ellipsoid, where nd represents the number of dimensions of the hyper-ellipsoid. Note that only the upper triangle (and diagonals) of the representative matrix is needed. On output, the lower triangle of ChoLowRepMatUpp (excluding diagonals) will be overwritten with the lower triangle of the Cholesky factorization of the input upper-triangle representative matrix, if and only if the argument ChoDia is present. Otherwise, the lower-triangle will remain intact.
[out]ChoDia: The diagonal elements of of the Cholesky factorization of the input upper-triangle representative matrix (optional).
[out]failed: A logical output which is .true. if the Cholesky factorization fails. (optional. If missing while an error occurs, the program will halt with error stop.)
[in]logVolUnitBall: The natural logarithm of the volume of a unit-radius ball in nd dimensions. (optional. default = genLogVolUnitBall(nd))
Returns
logVolEllipsoid : The logarithm of the volume of an nd-dimensional hyper-ellipsoid.
Warning
If the Cholesky factorization fails within this function and the ChoDia provided as output argument, then the first element of ChoDia will be set to a negative number
Remarks
There is really no reason to use this function for HPC solutions, in particular, when the function is called repeatedly with the same input covUpp. Each call of this function requires a full Cholesky factorization of the input Covariance Matrix which is computationally expensive. Use genLogVolEllipsoid_0D() when the Cholesky factorization is already precomputed and available.

◆ isInsideEllipsoid()

pure logical function GeometryEllipsoid_mod::isInsideEllipsoid ( integer(IK), intent(in)  nd,
real(RK)  NormedPoint,
real(RK)  InvRepMat 
)

Return .true. if the input NormedPoint (normalized with respect to the center of the target ellipsoid) is within or on the boundary of the ellipsoid whose boundary is described by the representative matrix \( \Sigma \) (RepMat), such that,.

\begin{equation} X^T ~ \Sigma^{-1} ~ X = 1 ~, \end{equation}

for all \(X\) on the boundary.

Parameters
[in]nd: The number of dimensions of the ellipsoid (the size of NormedPoint).
[in]NormedPoint: The input point, normalized with respect to the center of the ellipsoid, whose location with respect to the ellipsoid boundary is to be determined.
[in]InvRepMat: The inverse of the representative matrix of the target ellipsoid.
Returns
isInsideEllipsoid : The logical value indicating whether the input point is inside or on the boundary of the target ellipsoid.
Remarks
Note that the input matrix is the inverse of RepMat: InvRepMat.
