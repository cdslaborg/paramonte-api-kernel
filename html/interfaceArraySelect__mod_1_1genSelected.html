Detailed Description

Generate and return the rankth smallest value in the input array by first sorting its elements (optionally only between the specified indices [left,right]). By default, the array will be sorted in ascending order, unless the user-specific isSorted() external function is supplied to define a custom sorting criterion.

Parameters
[in]Array: The input contiguous array of shape (:) of either
  type real of kind RK32, RK64, or RK128, or
  type integer of kind IK8, IK16, IK32, or IK64, or
  type character of default kind SK, or
  containing the potentially unsorted sequence of values that is to be sorted and whose rankth smallest value is to be reported as the output selection.
[in]rank: The input integer of default kind IK, representing the index of the rankth smallest value in the input Array or the rankth ordered value according to the user-supplied external logical function isSorted().
isSorted: The optional, external user-specified function that takes two input scalar arguments of the same type and kind as the input Array. It returns a scalar logical of default kind LK that is .true. if the first input scalar argument is sorted with respect to the second input argument according to the user-defined condition within isSorted, otherwise, it is .false.. If Array is a Fortran string (i.e., an assumed-length scalar character), then both input arguments to isSorted() are character(1,SK) of default kind SK.

Warning
If left is present as an input argument, its value must be larger than 0 and less than right or its default value. Furthermore, the specified rank must be larger than or equal to the specified value for left. These conditions are only checked for when the preprocessor macro CHECK_ENABLD=1 is set when building the ParaMonte library.
If right is present as an input argument, its value must be less than or equal to the length of Array and larger than right or its default value. Furthermore, the specified rank must be smaller than or equal to the specified value for right. These conditions are only checked for when the preprocessor macro CHECK_ENABLD=1 is set when building the ParaMonte library.
The functions under this generic interface are impure when the external input argument isSorted() is present.
This pure procedures documented herein become impure when the ParaMonte library is compiled with preprocessor macro CHECK_ENABLED=1. By default, pure procedures are pure in release build and impure in debug and testing builds.
